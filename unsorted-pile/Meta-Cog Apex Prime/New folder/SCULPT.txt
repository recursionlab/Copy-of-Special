SCULPT: Systematic Tuning of Long Prompts
Objective
SCULPT is designed to optimize extended, unstructured prompts by reorganizing them into a hierarchical tree format and iteratively refining them using an actor–critic mechanism. The aim is to boost the performance of long-chain reasoning tasks by systematically addressing structural inconsistencies and errors.

Core Components
1. Prompt Structuring
Purpose: Transform flat, unstructured prompts into a hierarchical tree (T = (N, E)), where:
N (Nodes): Represent elements such as headings, instructions, and examples.
E (Edges): Define the parent–child relationships between these elements.
Benefit: This restructuring enables targeted, node-specific modifications without disrupting unrelated sections, ensuring the overall coherence of the prompt.
2. Critic Module
Role: Evaluate the current structured prompt and provide reflective feedback.
Feedback Types:
Preliminary Assessment (R_prelim): Reviews overall prompt clarity, structure, and completeness before task execution.
Error Assessment (R_error): Analyzes model output discrepancies on a validation set to pinpoint problematic nodes and suggest specific improvements.
Aggregation: Uses both similarity-driven and node-reference aggregation to capture recurring error patterns while avoiding overfitting to individual instances.
3. Actor Module
Role: Convert the Critic’s feedback into concrete actions to refine the prompt.
Process:
Operate on an induced subtree (T_induced) that contains only the sections needing modification.
Apply actions such as:
Structural Reordering
Instruction Update
Example Addition/Deletion/Refinement
Node Pruning/Expansion/Merging
Output: The modified tree is then linearized back into an updated text prompt (P′).
4. Beam Search with UCB-based Selection
Mechanism:
Maintain a beam of candidate prompts (B_t) during each iteration.
Use an Upper Confidence Bound (UCB) strategy to balance exploration (testing new variations) and exploitation (refining high-performing candidates).
Outcome: Select the top candidates that maximize a defined performance metric (e.g., accuracy, coherence) on a validation dataset.
Optimization Process
Initialization:

Start with the initial unstructured prompt (P₀) and transform it into the hierarchical tree format using the transformation function (t_init).
Iterative Refinement (for t = 1 to T):

Critic Evaluation:
Generate Preliminary Assessment and Error Assessment reflections.
Aggregate error feedback to identify common issues across nodes.
Actor Actions:
Apply targeted modifications (actions A) to the affected nodes.
Update the prompt tree (T′) and reassemble it into a new prompt candidate (P′).
Beam Search:
Evaluate updated prompts on a validation set using UCB scores.
Retain the top K candidates for the next iteration.
Convergence:

Continue iterations until performance plateaus (observed around 8 steps) or a maximum step count is reached.
Output the final optimized prompt that maximizes the performance metric Q on the validation set.
