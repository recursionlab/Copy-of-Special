# I. Core Separation

You are now distinguishing between two fundamentally different evaluators:

|Evaluator|Question|
|---|---|
|**Load-Bearing**|“Does this change system behavior?”|
|**Logic**|“Is this internally valid under transformation rules?”|

These are not equivalent.

A thing can be:

- logical but non-load-bearing,
    
- load-bearing but pre-logical,
    
- logical yet dynamically irrelevant,
    
- structurally necessary before formal logic even emerges.
    

This distinction is extremely important.

---

# II. Load-Bearing Is Ontological / Dynamical

Load-bearing concerns:

[  
\boxed{  
\text{causal structural necessity}  
}  
]

A component is load-bearing iff removing it changes:

- phase evolution,
    
- invariants,
    
- observables,
    
- transport,
    
- stability,
    
- geometry,
    
- recursion behavior.
    

This is fundamentally:

- dynamical,
    
- geometric,
    
- operational,
    
- energetic,
    
- topological.
    

Not symbolic.

Not deductive.

Not necessarily linguistic.

---

## Formal Criterion

A component (C) is load-bearing iff:

[  
\exists \mathcal O :  
\mathcal O(S)  
\neq  
\mathcal O(S \setminus C)  
]

for some observable (\mathcal O).

Meaning:  
removal changes measurable system structure.

---

# III. Logic Is Constraint-Coherent Transformation

Logic concerns:

[  
\boxed{  
\text{validity-preserving transformation}  
}  
]

It asks:

- does implication preserve consistency?
    
- do transformations commute correctly?
    
- are inference rules stable?
    
- do contradictions propagate?
    

Logic governs relations between statements or transformations.

Not necessarily existence.

Not necessarily dynamics.

---

## Formal Criterion

A logical system preserves:

[  
A \vdash B  
]

under admissible transformation rules.

Meaning:  
if premises hold, conclusions follow.

This is syntactic/semantic coherence.

Not ontological necessity.

---
