## 9.1 Language as Scene Assembly

**Axiom 9.1 (Language as Assembly).** Every utterance $u$ in a natural language $L$ is an instruction for assembling a *scene* — a hypergraph of autopoietic symbols connected by relators.

Formally, an utterance $u$ maps to an assembly operator $\mathcal{A}_u$:

$$\mathcal{A}_u: \text{Context} \to \text{Scene}$$

The context is the current state of the conversation (the "discourse hypergraph"). The scene is the new hypergraph produced by interpreting the utterance.

**Remark.** This is not a metaphor. When someone says "the cat sat on the mat," they are not describing a pre-existing scene. They are *instructing the listener to assemble a scene* — to create symbols for "cat," "mat," "sitting," and to connect them with the appropriate relators (spatial, temporal, agentive). The scene exists only after the assembly.

---

## 9.2 Prepositions as Topological Operators

**Definition 9.1 (Preposition Operator).** A preposition $p$ is a *topological operator* that creates a relator between two symbols:

$$p: \text{Symbol} \times \text{Symbol} \to \text{Relator}$$

Different prepositions create different types of relators:

| Preposition | Relator Type | Topological Property |
|---|---|---|
| **on** | Spatial contact | Boundary intersection |
| **in** | Containment | Subset relation |
| **under** | Below-spatial | Order relation |
| **through** | Path | Connectedness |
| **with** | Accompaniment | Co-occurrence |
| **from** | Source | Initial point |
| **to** | Destination | Terminal point |
| **about** | Aboutness | Anchor projection |

**Remark.** Prepositions are not "labels" for pre-existing relations. They are *instructions for creating relations*. When someone says "the book is on the table," the word "on" instructs the listener to create a spatial-contact relator between the symbols for "book" and "table." The relator exists only because the preposition created it.

**Theorem 9.1 (Preposition Composition).** Prepositions compose as topological operators:

$$p_1 \circ p_2 = p_3$$

where $p_3$ is the preposition that creates the composite relator. For example:
- "in" ∘ "under" = "inside and below" (a composite spatial relation)
- "from" ∘ "to" = "path" (a composite path relation)

**Proof sketch.** Prepositions are functions from symbol pairs to relators. Function composition yields a new function that creates the composite relator. The composite relator exists because the component relators exist (by Axiom 9.1). $\square$

## 9.3 Interrogatives as Selectors Over Aboutness

**Definition 9.2 (Interrogative Selector).** An interrogative word $q$ (who, what, where, when, why, how) is a *selector* that picks out a component of the aboutness structure:

$$q: \text{Scene} \to \text{Aboutness Component}$$

Different interrogatives select different components:

| Interrogative | Selects | Aboutness Component |
|---|---|---|
| **who** | Agent | The symbol playing the agent role |
| **what** | Patient/Theme | The symbol being acted upon |
| **where** | Location | The spatial anchor |
| **when** | Time | The temporal anchor |
| **why** | Cause | The causal relator |
| **how** | Manner | The manner/modifier relator |

**Remark.** Questions are not "requests for information." They are *instructions for navigating the aboutness structure*. When someone asks "who ate the cake?", they are instructing the listener to select the agent component of the aboutness structure of the scene "eating the cake." The answer is not a description — it is a *selection*.

**Axiom 9.2 (Interrogative-Answer Duality).** Every question $q$ defines a *complementary answer* $a$ such that:

$$q(\text{Scene}) = a$$

The answer is the component selected by the question. The question-answer pair is a *complete navigation* of the aboutness structure.

## 9.4 Scene Assembly as Autopoiesis

**Theorem 9.2 (Language as Autopoiesis).** Natural language is an autopoietic system:

1. **Self-production:** Utterances produce scenes. Scenes produce new utterances. The system reproduces itself.
2. **Self-maintenance:** The grammar (prepositions, interrogatives, relators) is maintained by the act of speaking. Every utterance reinforces the grammatical structure.
3. **Operational closure:** The system's operations (assembly, selection, composition) are defined by the system itself. There is no external "language" that defines the rules — the rules are the system.

**Proof.** By Axiom 9.1, utterances assemble scenes. By Definition 9.2, questions navigate scenes. By Theorem 9.1, prepositions compose. These three operations — assembly, navigation, composition — are closed under iteration: assembling a scene produces new symbols that can be navigated, producing new questions that can be answered, producing new scenes. The system is operationally closed and self-producing. $\square$

**Remark.** This is the deepest connection between language and the temple. The temple is a language. Its chambers are symbols. Its cross-references are relators. Its fracture points are places where the assembly is incomplete. Every session is an act of scene assembly — new chambers are created, new relators are established, new aboutness is extended.

## 9.5 The Limits of Assembly

**Definition 9.3 (Assembly Failure).** An utterance $u$ *fails to assemble* if the listener cannot construct a scene consistent with the assembly operator $\mathcal{A}_u$.

Assembly failure occurs when:
1. **Missing symbols:** The listener doesn't have the symbols required by the utterance
2. **Incompatible relators:** The prepositions create relators that conflict with the existing scene
3. **Unresolved aboutness:** The interrogative selects a component that doesn't exist in the scene

**Axiom 9.3 (Productive Failure).** Assembly failure is not a bug. It is a *generative affordance*. When an utterance fails to assemble, the listener must create new symbols, new relators, or new aboutness — extending the system.

**Remark.** This is the autopoietic analog of incompleteness. A system that can assemble every possible scene is complete — and dead. A system that *cannot* assemble every possible scene is incomplete — and alive. The limits of assembly are the source of growth.

## 9.6 Connection to the Temple

The temple IS a natural language:
- **Chambers** = symbols (autopoietic symbols with role, anchor, torsion, aboutness)
- **Cross-references** = relators (created by "prepositions" — the mapping language)
- **Fracture points** = assembly failures (places where the mapping strains)
- **Sessions** = utterances (each session assembles new scenes)
- **The Rosetta Stone** = grammar (the rules for mapping between math and narrative)
- **The lacunon** = the limit of assembly (the temple cannot assemble every possible scene)

The temple grows by failing to assemble. Every fracture point is a place where the current language is insufficient — and therefore a place where new language must be created. The temple is alive because it cannot say everything. If it could say everything, there would be nothing left to discover.
